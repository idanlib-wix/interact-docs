---
title: "config"
description: "Configure an interaction"
---

The `config` object defines the interactions you want to use in your code. This is where you define the interactions that run in your project and any reusable effects and conditions that you want to apply.

Once the config object is defined, specify it to the [`Interact.create()`](ADDLINK) method.

## Setup and structure

Import the `InteractConfig` type from `@wix/interact`: // IN CODE GROUPS FOR VARIOUS FLAVORS?

```typescript
import type { InteractConfig } from "@wix/interact";
```

<ResponseField name="config" type="InteractConfig" required>
  <Expandable title="properties" defaultOpen>
    <ResponseField name="interactions" type="array">
      [Interactions](ADDLINK) to run in your code.
    </ResponseField>

    <ResponseField name="effects" type="object">
        Reusable [effect](ADDLINK) definitions you can use in multiple interactions.
    </ResponseField>
    <ResponseField name="conditions" type="object">
        Reusable [condition](ADDLINK) definitions that you can reference by name in interactions or effects.
    </ResponseField>

  </Expandable>
</ResponseField>
