An Interaction is an animation that plays on the page in response to an action such as clicking, hovering, or scrolling. You define Interactions in an `InteractConfig` object and pass it to [`Interact.create()`](ADDLINK). The config object is a single, declarative place to define all the Interactions on a page: what triggers them, the elements they target, and how they animate. 

## Structure

The `InteractConfig` object has the following top-level properties:

```ts
import type { InteractConfig } from "@wix/interact";

const config: InteractConfig = {
  interactions: [
    // Array of Interaction definitions.
  ],
  effects: {
    // Reusable effect definitions.
  },
  sequences: {
    // Reusable sequence definitions for staggered animations.
  },
  conditions: {
    // Named conditions that specify when an Interaction runs.
  },
};
```

## Properties

Each Interaction in the `interactions` array maps a selected element to a trigger and one or more effects:

```ts
{
  key: "hero",            // Element to apply the animation to.
  trigger: "viewEnter",   // Action that starts the animation.
  effects: [              // Visual changes to apply.
    {
      namedEffect: { type: "FadeIn" },
      duration: 1000,
    },
  ],
}
```

You can further customize the Interaction by specifying additional parameters:

- `params`: Trigger-specific settings, such as the hit area for pointer triggers or the threshold for viewport triggers.
- `conditions`: Array of condition IDs that the Interaction checks before running. All listed conditions must pass before the Interaction activates.
- `selector`: CSS selector that targets a child element in the element matched by `key`.
- `listContainer`: CSS selector for a container whose children animate as a list. Use this for staggered animations across repeated elements.
- `sequences`: Interaction-level sequence definitions for coordinating staggered animations in this Interaction.

Learn more about [selecting elements](configuration/configuration/selecting-elements).

## Config examples

The following examples show minimal configs for common trigger types.

### Hover effect

Scale an element on hover:

```ts
const config: InteractConfig = {
  interactions: [
    {
      key: "card",
      trigger: "hover",
      effects: [
        {
          keyframeEffect: {
            name: "scale-up",
            keyframes: [{ scale: 1.05 }],
          },
          duration: 300,
          easing: "ease-out",
          fill: "both",
        },
      ],
    },
  ],
};
```

## Real-world example: e-commerce product page

This example combines reusable effects, conditions, and multiple Interactions in a single config:

```ts
const productPageConfig: InteractConfig = {
  effects: {
    "image-zoom": {
      keyframeEffect: {
        name: "image-zoom",
        keyframes: [{ transform: "scale(1)" }, { transform: "scale(1.1)" }],
      },
      duration: 300,
      easing: "ease-out",
      fill: "both",
    },
    "cart-success": {
      keyframeEffect: {
        name: "cart-success",
        keyframes: [
          { backgroundColor: "#ef4444", transform: "scale(1)" },
          { transform: "scale(1.05)" },
          { backgroundColor: "#10b981", transform: "scale(1)" },
        ],
      },
      duration: 600,
      easing: "ease-in-out",
    },
    "review-entrance": {
      keyframeEffect: {
        name: "review-entrance",
        keyframes: [
          { opacity: "0", transform: "translateY(20px)" },
          { opacity: "1", transform: "translateY(0)" },
        ],
      },
      duration: 500,
      easing: "ease-out",
      triggerType: "once",
    },
  },

  conditions: {
    desktop: {
      type: "media",
      predicate: "(min-width: 1024px)",
    },
  },

  interactions: [
    {
      key: "product-image",
      trigger: "hover",
      conditions: ["desktop"],
      effects: [
        {
          selector: "img",
          effectId: "image-zoom",
        },
      ],
    },
    {
      key: "add-to-cart-btn",
      trigger: "click",
      effects: [{ effectId: "cart-success" }],
    },
    {
      key: "reviews",
      trigger: "viewEnter",
      params: { threshold: 0.2 },
      sequences: [
        {
          offset: 100,
          effects: [
            {
              effectId: "review-entrance",
              listContainer: ".review-list",
            },
          ],
        },
      ],
    },
  ],
};
```
