Named effects are pre-made animations from the `@wix/motion-presets` package that let you add polished animations to your Interactions without writing custom keyframes.

## Available named effects

Named effects are organized by category, each designed for a specific trigger type.

### Entrance effects

Entrance effects are single-shot reveal animations triggered by the `viewEnter` trigger. These effects play once when an element enters the viewport:

- `ArcIn`
- `BlurIn`
- `BounceIn`
- `CurveIn`
- `DropIn`
- `ExpandIn`
- `FadeIn`
- `FlipIn`
- `FloatIn`
- `FoldIn`
- `GlideIn`
- `RevealIn`
- `ShapeIn`
- `ShuttersIn`
- `SlideIn`
- `SpinIn`
- `TiltIn`
- `TurnIn`
- `WinkIn`

### Ongoing effects

Ongoing effects are looping animations compatible with all trigger types. Use them for continuous attention effects:

- `Bounce`
- `Breathe`
- `Cross`
- `DVD`
- `Flash`
- `Flip`
- `Fold`
- `Jello`
- `Poke`
- `Pulse`
- `Rubber`
- `Spin`
- `Swing`
- `Wiggle`

### Scroll effects

Scroll-driven animations are triggered by the `viewProgress` trigger. The element's scroll position controls the animation progress:

- `ArcScroll`
- `BlurScroll`
- `FadeScroll`
- `FlipScroll`
- `GrowScroll`
- `MoveScroll`
- `PanScroll`
- `ParallaxScroll`
- `RevealScroll`
- `ShapeScroll`
- `ShrinkScroll`
- `ShuttersScroll`
- `SkewPanScroll`
- `SlideScroll`
- `Spin3dScroll`
- `SpinScroll`
- `StretchScroll`
- `TiltScroll`
- `TurnScroll`

<Warning>
Scroll named effects require a `range` option set to `'in'`, `'out'`, or `'continuous'`. Omitting `range` can cause unexpected behavior.
</Warning>

### Background scroll effects

Scroll-driven background animations triggered by the `viewProgress` trigger. They apply parallax and transformation effects to background elements.

- `BgCloseUp`
- `BgFade`
- `BgFadeBack`
- `BgFake3D`
- `BgPan`
- `BgParallax`
- `BgPullBack`
- `BgReveal`
- `BgRotate`
- `BgSkew`
- `BgZoom`
- `ImageParallax`

### Mouse effects

Pointer-driven animations triggered by `pointerMove`. They respond to cursor movement over elements or the viewport:

- `AiryMouse`
- `BlobMouse`
- `BlurMouse`
- `BounceMouse`
- `CustomMouse`
- `ScaleMouse`
- `SkewMouse`
- `SpinMouse`
- `SwivelMouse`
- `Tilt3DMouse`
- `Track3DMouse`
- `TrackMouse`

## Set up a named effect

To use a named effect, you need to:

<Steps>
  <Step title="Install motion presets">

    Install the `@wix/motion-presets` package:

    ```bash
    npm install @wix/motion-presets
    ```

  </Step>
  <Step title="Register effects">
    Import the named effects you want to use, and register them with Interact. You can import them selectively or register all available effects:

    ```typescript
    // Selective import (recommended for smaller bundles)
    import { FadeIn, BounceIn } from "@wix/motion-presets";
    Interact.registerEffects({ FadeIn, BounceIn });
    ```

    ```typescript
    // Full import (registers all named effects)
    import * as presets from "@wix/motion-presets";
    Interact.registerEffects(presets);
    ```

    <Note>
    You must call [`registerEffects()`](ADDLINK) before calling [`Interact.create()`](ADDLINK).
    </Note>

  </Step>
  <Step title="Use a named effect in your config">
    In the Interaction's `effects` array, set the `type` property inside the `namedEffect` object to the name of the effect you want to use:

    <Tabs>
      <Tab title="React">

      ```tsx
      import React, { useEffect } from "react";
      import { Interact, Interaction, InteractConfig } from "@wix/interact/react";
      import { FadeIn } from "@wix/motion-presets";

      Interact.registerEffects({ FadeIn });

      const config: InteractConfig = {
        interactions: [
          {
            key: "hero",
            trigger: "viewEnter",
            effects: [
              {
                namedEffect: { type: "FadeIn" },
                duration: 1000,
              },
            ],
          },
        ],
      };

      function App() {
        useEffect(() => {
          const instance = Interact.create(config);
          return () => instance.destroy();
        }, []);

        return (
          <Interaction tagName="div" interactKey="hero">
            <h1>Welcome</h1>
          </Interaction>
        );
      }

      export default App;
      ```

      </Tab>
      <Tab title="Web Components">

      ```typescript
      import { Interact } from "@wix/interact/web";
      import { FadeIn } from "@wix/motion-presets";

      Interact.registerEffects({ FadeIn });

      const config = {
        interactions: [

            key: "hero",
            trigger: "viewEnter",
            effects: [
              {
                namedEffect: { type: "FadeIn" },
                duration: 1000,
              },
            ],
          },
        ],
      };

      Interact.create(config);
      ```

      </Tab>
      <Tab title="JavaScript">

      ```typescript
      import { Interact } from "@wix/interact";
      import { FadeIn } from "@wix/motion-presets";

      Interact.registerEffects({ FadeIn });

      const config = {
        interactions: [
          {
            key: "hero",
            trigger: "viewEnter",
            effects: [
              {
                namedEffect: { type: "FadeIn" },
                duration: 1000,
              },
            ],
          },
        ],
      };
      ```

## Customize a named effect

Many named effects accept optional parameters that let you control how the animation looks and behaves. Available parameters vary by category:

| Category | Parameter | Description | Example values |
| -------- | --------- | ----------- | -------------- |
| Entrance | `direction` | Where the animation originates. | `"top"`, `"bottom"`, `"left"`, `"right"` |
| Entrance | `power` | Intensity of the effect. | `"soft"`, `"medium"`, `"hard"` |
| Scroll | `range` | When the animation plays relative to scroll position (required). | `"in"`, `"out"`, `"continuous"` |
| Scroll | `speed` | Movement speed. | `0.5`, `1`, `2` |

Each named effect supports different parameters. Learn more about [named effects](ADDLINK).

To customize a named effect, specify the named effect's `type` along with the parameters you want to customize. For example, this custom effect `FadeIn` starts softly from the left:

```typescript
// Inside your config's interactions[].effects array
effects: [
  {
    namedEffect: {
      type: "FadeIn",
      direction: "left",
      power: "soft",
    },
    duration: 800,
  },
];
```
